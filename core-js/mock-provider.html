Mock Provider

Overview​

The mock provider is an in-memory implementation of FeatureProvider. It is designed for testing and local development — no external service is required.

All flag evaluations and config lookups are resolved from plain objects you pass at construction time. Every operation is synchronous under the hood, wrapped in Promise.resolve for interface compatibility.

Setup​

npm install @feature-gate/core

No additional dependencies are needed. The mock provider is bundled with the package.

Usage Example​

import { featureManagement, mockProvider } from '@feature-gate/core';

const provider = mockProvider({
  flags: {
    'new-checkout': true,
    'dark-mode': false,
  },
  configs: {
    'price-rounding': 2,
    'banner-text': 'Welcome back!',
  },
});

await provider.initialize();

const service = featureManagement(provider);

const enabled = service.isEnabledSync({ flag: 'new-checkout' });
// → true

const price = service.getConfigSync({ key: 'price-rounding', fallback: 0 });
// → 2

API Reference​

mockProvider(config?)​

Returns a FeatureProvider backed by in-memory maps.

import { mockProvider, MockProviderConfig } from '@feature-gate/core';

const provider = mockProvider(config?: MockProviderConfig);

MockProviderConfig​

FieldTypeRequiredDescription
flagsRecord<string, boolean>NoMap of flag names to their boolean values.
configsRecord<string, unknown>NoMap of config keys to their values.

Both fields default to empty objects when omitted.

Behavior​

Method / PropertyReturns
initializedfalse before initialize(), true after.
initialize()Sets initialized = true. Resolves immediately (no-op otherwise).
isEnabledSync({ flag })Returns flags[flag] ?? false.
areEnabledSync({ flags })Returns a boolean array, one entry per flag.
getConfigSync({ key, fallback })Returns configs[key] ?? fallback.
isEnabled(...)Async wrapper around isEnabledSync.
areEnabled(...)Async wrapper around areEnabledSync.
getConfig(...)Async wrapper around getConfigSync.
shutdown()Resolves immediately (no-op).

Missing flags resolve to false. Missing configs resolve to the provided fallback.

Testing Patterns​

Test each flag state independently​

Create one mockProvider per test case (or beforeEach) to keep tests isolated and avoid shared state:

import { describe, it, expect, beforeEach } from 'vitest';
import { featureManagement, mockProvider } from '@feature-gate/core';

describe('checkout page', () => {
  it('shows new checkout when flag is on', async () => {
    const provider = mockProvider({ flags: { 'new-checkout': true } });
    await provider.initialize();
    const service = featureManagement(provider);

    expect(service.isEnabledSync({ flag: 'new-checkout' })).toBe(true);
  });

  it('shows legacy checkout when flag is off', async () => {
    const provider = mockProvider({ flags: { 'new-checkout': false } });
    await provider.initialize();
    const service = featureManagement(provider);

    expect(service.isEnabledSync({ flag: 'new-checkout' })).toBe(false);
  });
});

Test config-driven behavior​

it('applies remote timeout config', async () => {
  const provider = mockProvider({ configs: { 'request-timeout-ms': 3000 } });
  await provider.initialize();
  const service = featureManagement(provider);

  const timeout = service.getConfigSync({ key: 'request-timeout-ms', fallback: 5000 });
  expect(timeout).toBe(3000);
});

Test the fallback path​

Omit a flag or config key to verify your code handles the off/missing case correctly:

it('falls back to default timeout when config is absent', async () => {
  const provider = mockProvider(); // no configs
  await provider.initialize();
  const service = featureManagement(provider);

  const timeout = service.getConfigSync({ key: 'request-timeout-ms', fallback: 5000 });
  expect(timeout).toBe(5000);
});

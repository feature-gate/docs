Custom Provider

Overview​

You can connect any feature flag backend to @feature-gate/core by implementing the FeatureProvider interface. This page walks through building a working custom provider from scratch and wiring it into featureManagement().

The FeatureProvider Interface​

import type {
  FeatureProvider,
  IsEnabledParams,
  AreEnabledParams,
  GetConfigParams,
} from '@feature-gate/core';

Your provider must implement these eight methods and one property:

Method / PropertySignatureDescription
initializedbooleanRead-only flag. Must be false until initialize() completes successfully. featureManagement() uses this to guard all service calls.
initialize() => Promise<void>Called by you before passing the provider to featureManagement(). Connect to your backend and set initialized = true here.
shutdown() => Promise<void>Called when the service is no longer needed. Close connections, flush buffers.
isEnabledSync(params: IsEnabledParams) => booleanSynchronous flag check using cached data.
areEnabledSync(params: AreEnabledParams) => boolean[]Synchronous check for multiple flags at once.
getConfigSync<T>(params: GetConfigParams<T>) => TSynchronous config read. Must return fallback when the key is missing.
isEnabled(params: IsEnabledParams) => Promise<boolean>Async flag check.
areEnabled(params: AreEnabledParams) => Promise<boolean[]>Async check for multiple flags.
getConfig<T>(params: GetConfigParams<T>) => Promise<T>Async config read.

Parameter types​

interface FeatureContext {
  userId?: string;
  sessionId?: string;
  ip?: string;
  environment?: string;
  properties?: Record<string, string>;
}

interface IsEnabledParams {
  flag: string;
  context?: FeatureContext;
}

interface AreEnabledParams {
  flags: string[];
  context?: FeatureContext;
}

interface GetConfigParams<T> {
  key: string;
  fallback: T;      // required — return this when the key is missing
  context?: FeatureContext;
}

Using baseProvider as a Starting Point​

If you only need to override a few methods, you can spread baseProvider instead of implementing the full interface. Any method you do not override will throw a clear "not implemented" error at runtime, so you notice quickly if a caller reaches a method you forgot.

import { baseProvider, featureManagement } from '@feature-gate/core';

const myFlags: Record<string, boolean> = {
  'new-checkout': true,
};

const provider = {
  ...baseProvider,

  isEnabledSync: ({ flag }) => myFlags[flag] ?? false,
  areEnabledSync: ({ flags }) => flags.map((f) => myFlags[f] ?? false),
  getConfigSync: ({ fallback }) => fallback,
};

await provider.initialize();

const service = featureManagement(provider);

initialize and shutdown are no-ops in baseProvider, so you only need to override them if you have a real connection to open or close.

Step-by-Step: Build a Custom Provider​

This example builds a provider that fetches flags from a simple HTTP JSON API.

Step 1 — Define your config type​

// my-provider.ts

export interface MyProviderConfig {
  apiUrl: string;
  apiKey: string;
  pollIntervalMs?: number; // optional: how often to refresh (default: 30 000)
}

Step 2 — Implement FeatureProvider​

import type {
  FeatureProvider,
  IsEnabledParams,
  AreEnabledParams,
  GetConfigParams,
} from '@feature-gate/core';

export const myProvider = (config: MyProviderConfig): FeatureProvider => {
  const { apiUrl, apiKey, pollIntervalMs = 30_000 } = config;

  // Local cache populated by initialize() and the poll loop
  let flags: Record<string, boolean> = {};
  let configs: Record<string, unknown> = {};
  let pollTimer: ReturnType<typeof setInterval> | undefined;

  // Fetch the latest rules from the backend
  const fetchRules = async (): Promise<void> => {
    const res = await fetch(`${apiUrl}/rules`, {
      headers: { Authorization: `Bearer ${apiKey}` },
    });
    if (!res.ok) throw new Error(`Failed to fetch rules: ${res.status}`);
    const data = await res.json();
    flags = data.flags ?? {};
    configs = data.configs ?? {};
  };

  let initialized = false;

  return {
    get initialized(): boolean { return initialized; },

    // ----------------------------------------------------------------
    // Lifecycle
    // ----------------------------------------------------------------

    async initialize(): Promise<void> {
      await fetchRules();
      pollTimer = setInterval(() => {
        fetchRules().catch(() => {/* errors are non-fatal during polling */});
      }, pollIntervalMs);
      initialized = true;
    },

    async shutdown(): Promise<void> {
      clearInterval(pollTimer);
    },

    // ----------------------------------------------------------------
    // Sync (use the in-memory cache)
    // ----------------------------------------------------------------

    isEnabledSync({ flag }: IsEnabledParams): boolean {
      return flags[flag] ?? false;
    },

    areEnabledSync({ flags: keys }: AreEnabledParams): boolean[] {
      return keys.map((k) => flags[k] ?? false);
    },

    getConfigSync<T>({ key, fallback }: GetConfigParams<T>): T {
      return key in configs ? (configs[key] as T) : fallback;
    },

    // ----------------------------------------------------------------
    // Async (same logic — extend here if you need fresh network calls)
    // ----------------------------------------------------------------

    async isEnabled(params: IsEnabledParams): Promise<boolean> {
      return this.isEnabledSync(params);
    },

    async areEnabled(params: AreEnabledParams): Promise<boolean[]> {
      return this.areEnabledSync(params);
    },

    async getConfig<T>(params: GetConfigParams<T>): Promise<T> {
      return this.getConfigSync(params);
    },
  };
};

Step 3 — Initialize and pass the provider to featureManagement()​

import { featureManagement } from '@feature-gate/core';
import { myProvider } from './my-provider';

const provider = myProvider({
  apiUrl: 'https://flags.example.com/api',
  apiKey: process.env.FLAGS_API_KEY!,
  pollIntervalMs: 60_000,
});

await provider.initialize();

const service = featureManagement(provider);

// Use the service
const isNewCheckout = service.isEnabledSync({
  flag: 'new-checkout',
  context: { userId: 'user-123' },
});

const timeout = service.getConfigSync<number>({
  key: 'request-timeout-ms',
  fallback: 5000,
});

// Shut down cleanly at process exit
process.on('SIGTERM', async () => {
  await service.shutdown();
  process.exit(0);
});

Key Rules​

Call initialize() before featureManagement(). You are responsible for calling provider.initialize() and awaiting it. Set initialized = true at the end of your initialize() implementation — this is the signal featureManagement() uses to decide if the service is ready. If initialize() throws, let the error propagate so the caller knows setup failed.

getConfigSync must always return T, never undefined. The interface contract requires a concrete value. When the key is missing, return fallback.

Sync methods must not do I/O. They are expected to be fast and synchronous. Cache what you need during initialize() and refresh it in a background poll.

Errors thrown from provider methods are caught by FeatureManagementService. The service wraps every call in a try/catch and returns safe fallbacks (false, undefined, []). You can let expected errors propagate — the caller will never see them.

shutdown() should be idempotent. It may be called more than once; guard against double-cleanup where necessary.
